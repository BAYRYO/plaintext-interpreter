MarkWeave - Documentation

Introduction

MarkWeave est un convertisseur de texte en HTML qui combine la simplicité du Markdown avec des fonctionnalités avancées de gestion des assets et de navigation dynamique.

Installation et Configuration

Prérequis

    {Python 3.x, pip (gestionnaire de paquets Python), Connexion internet pour les CDN}

Installation

pip install markweave

Configuration

Le fichier de configuration principal se trouve dans configs/config.yml. Il permet de personnaliser :

    {Encodage des fichiers, Chemins des templates, Styles de navigation, Patterns regex, Messages système}

Utilisation

Commandes de Base

[[Commande|Description|Exemple]] [[python main.py input.html output.html|Conversion simple d'un fichier|python main.py docs/input.html build/output.html]] [[python main.py --live input.html output.html|Mode édition en direct|python main.py --live docs/guide.html build/guide.html]] [[python main.py --config custom_config.yml input.html output.html|Utilisation d'une config personnalisée|python main.py --config my_config.yml input.html output.html]]

Syntaxe Supportée

MarkWeave prend en charge les éléments HTML suivants :

    {Titres (h1-h6), Paragraphes, Listes ordonnées et non-ordonnées, Blocs de code, Tableaux}

Fonctionnalités Spéciales

L'application offre plusieurs fonctionnalités avancées :

    {Navigation dynamique automatique, Coloration syntaxique du code, Mode édition en direct, Gestion automatique des assets}

Mode Édition en Direct

Le mode édition en direct (--live) permet de :

    {Visualiser les changements en temps réel, Rafraîchissement automatique du navigateur, Prévisualisation instantanée du rendu}

Gestion des Assets

Structure des Assets

templates/assets/
    ├── css/
    │   ├── style.css
    │   └── icons.css
    ├── js/
    │   └── script.js
    └── images/
        ├── favicon.ico
        └── favicon.svg

Configuration des Assets

Les assets sont configurés dans config.yml et sont automatiquement gérés lors de la génération du HTML.

Personnalisation

Templates

Les templates principaux sont :

    {base.html - Template principal, header.html - En-tête de page, footer.html - Pied de page, navigation.html - Panneau de navigation}

Styles

La personnalisation des styles peut se faire via :

    {Modification des fichiers CSS existants, Configuration des paramètres dans config.yml, Ajout de nouveaux fichiers CSS}

Dépannage

Les logs de l'application sont disponibles dans logs/converter.log et incluent des informations détaillées sur :

    {Erreurs de conversion, Problèmes de configuration, Activité du mode édition en direct}
